Creating Unique Binary Vectors of Length n (in R)

9 February 2017

I found myself recently working on a computational-modeling project whereby I needed to train a neural network on a series of unique binary vectors of length 10; that is, I needed to create a linearly independent set of vectors. It turns out that there is a simple solution in R that I wanted to document here, both for posterity and to help out anyone who'd need to do something similar for their own project.

First, install the hier.part package in R. Run the following command to install that package.

install.packages("hier.part")

With the package now installed, let's load the package, which can be achieved with either the require() or library() commands (see this for an interesting argument against using the require() command).

require(hier.part)

Note that we'll be using the combos() command from this library. To list all available functions in a library, simply run the following code and replace "hier.part" with the name of the package with which you're working:

library(help = hier.part)

Let's now create an object in which to store our uniquely permuted vectors of length 10. (Note that if you want a set of unique vectors of lengths other than 10, simply replace 10 with the desired length.) The following command achieves that for us:

binary = combos(10)$binary

Next, let's convert our binary object into a data frame. This step was useful for me because ultimately I needed to produce a structure—which contained the unique vectors—over which I could write a for() loop.

binary.dataframe=data.frame(binary)

Finally, let's convert that binary.dataframe into a .csv file and export it using the write() command.

write.csv(binary.dataframe, file="binary.csv")

That is it! I hope someone finds this short tutorial useful.

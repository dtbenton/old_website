On Nested For() Loops (in R)

24 February 2017

Due perhaps to their robustness, two of the more indispensible concepts and tools in programming and statistics are the for() loop and the nested (double or triple) for() loop. These are methods that are used to repeat a block of code a specified number of times. Even though R is technically a vectorized language (i.e., an array programming language) and thus doesn't require the use of complicated for() loops to achieve desired operations, becoming familiar with the concept of the for() loop and nested for() loops cannot (and should not) be understated. Indeed, this understanding is particularly important when learning languages that do not make use of vectorization such as Python, C, JavaScript, Objective-C, etc.. My aim in this (admittedly) brief post is to discuss the idea of the nested for() loop in the context of a simple and reproducible example.

As we have done in previous posts, we'll begin by defining two variables. We'll create an age and sex variable, like so.

The key idea here, in terms of why R outputs what it does, is that when R (or any other language for that matter) executes a nested for() loop, first it executes the innermost loop and applies a desired function (e.g., print) to every element in the innermost list, array, etc. and then executes the outermost loop and applies some desired function to each element in that list in an iterative manner. Critically, while the innermost function is being applied to the elements of the innermost list n times—where 'n' refers to the number of elements in the list or array—R will apply the same function to the outermost element until the inside function has been exhaustively applied.

Once the function has been applied to every element of the innermost list, R moves to the second element of the outermost list, applies the desired function to it, and then applies the innermost function to the innermost list until the function has been applied to every element in that list. To be sure, R will continue to apply the outermost function to the second element of the outermost list so long as the innermost function is being applied to every element in the innermost list or array. This process then repeats itself, with R moving to the third element of the outermost list and so on. Thus, from above, R prints out the first element of age (20) along with the first element of sex (male), after which it again prints out the first element of age (20) along with the second element of sex (female). Given that there are only two elements in sex, next R prints out the second element of age (24) along with the first element of sex (male), after which the second element of age (24) is printed alongside the second element of sex (female). Finally, R prints out the third element in age (28) along with the first element of sex (male), followed by the third element of age (28) along with the second element of sex (female). Notice that what R has done here is to create every combination of age and sex. It should be clear why.

Violà—and that's it! If you followed along with my discussion of why R printed out what it did given the above nested for() loop, it should be easy to extend this idea to more complicated loops. If you're still unclear, I'd encourage you to reread this post or use online resources.
